# 開発理由
<p align="center">
<img width="600"  src="https://github.com/user-attachments/assets/b8cdb34a-87d5-41e8-95c6-182aca2ecd96" />
</p>
 きっかけは友達と集まって課題をしようと思った時です。まとまって集まれる場所として授業に使われていない教室を利用することはよくありました。しかし、
私たち学生では次の時間に今居る教室が授業に使われるかどうかを事前に知ることができず、次の時間までに授業が始まる雰囲気を察知して退出すると、手間でした。

まず考えたのは「学生も教室予約が可能」でした。大学には公式の教室予約システムがあり、いち学生の力ではこれに干渉することは現実的ではないと考えました。
そこで、「既存の教室予約システムが反映された、教室の予約状況の一覧が見れるサイトを作成し、使われていない教室を私たちが予約できるサイト」を作成するのはどうだろうか？
これが私が最初にこのサイトを制作物として作成しようとしたきっかけです。

# 開発環境と作成過程
## 作成計画
Chat GPT o3 miniに作成を依頼しました。
細かい仕様や多くの内容を最初から伝えるとうまくコードが出力されないことがわかりました。そこで、大枠の作成を命令し、
完成されたコードを渡しながら、少しづつ要求を増やしていくようにしました。

## 使用言語とシステム概要
### 言語
**内部はC言語、出力はHTML**

大学ではC言語を学習していましたので、コードの理解ができるように、主要な内容はC言語で作成しました。当時の私の限られた知識では、データ管理が容易にする方法を考えた時、各教室の情報は構造体で管理した方がいいと思いました。そして別のシステムから構造体群に格納されているデータにアクセスするようにしました。Cは高速演算が可能であることも魅力でしたので、サイトがサクサク動くのも期待しました。

各教室クリックすると、構造体で管理された情報が掲載されます。「教室の詳細な設備」はごちゃごちゃするため、折りたためるようにしました。混み具合も数字がマス目に並ぶを見づらくなるので、色も追加して、パッとわかるようにしました。この画面に限らず、すべての操作画面と操作方法において「シンプルで見やすいデザインでありながら、欲しい情報はすぐ視覚的にもすぐわかる。」を目指して一番力を入れて時間を捧げました。
